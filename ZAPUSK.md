# Запуск «Зеленое образование» — инструкция для всех

Ничего настраивать вручную **не нужно**: в архиве уже есть Docker-конфиг, готовые переменные окружения и скрипт, который всё поднимает сам.

---

## Что нужно один раз установить на компьютер

| Программа | Зачем | Ссылка |
|-----------|--------|--------|
| **Node.js LTS** (20 или 22) | запуск сайта | https://nodejs.org/ |
| **Docker Desktop** | база данных в контейнере | https://www.docker.com/products/docker-desktop/ |

После установки Docker **запустите Docker Desktop** и дождитесь, пока он полностью загрузится (иконка в трее без «Starting»).

Проверка (необязательно), в PowerShell:

```powershell
node -v
docker --version
```

---

## Запуск проекта (каждый раз)

### Способ 1 — самый простой (Windows)

1. Распакуйте архив `GreenEdu-latest.zip` в любую папку, например `C:\Projects\GreenEdu`.
2. Убедитесь, что **Docker Desktop запущен**.
3. Дважды щёлкните файл **`ЗАПУСК.bat`** в папке проекта.

Скрипт сам:

- скопирует готовые настройки в `.env`;
- скачает и запустит PostgreSQL в Docker (первый раз скачает образ — 1–3 минуты);
- установит зависимости (`npm install`, только при первом запуске);
- создаст таблицы в базе;
- создаст **демо-пользователя**;
- запустит сайт и откроет браузер.

Адрес сайта: **http://localhost:3000**

Остановить: закройте чёрное окно терминала или нажмите **Ctrl+C**.

---

### Способ 2 — через терминал (Windows / macOS / Linux)

```bash
cd путь/к/GreenEdu
npm run go
```

Та же автоматическая настройка + запуск сайта.

После запуска можно сразу войти **без регистрации**:

| Поле | Значение |
|------|----------|
| Email | `demo@demo.ru` |
| Пароль | `demo1234` |

Или нажмите **Регистрация** и создайте свой аккаунт (пароль от 6 символов).

После входа в меню появится **Курсы** — лекции в PDF.

## Повторный запуск (на следующий день)

1. Запустите **Docker Desktop**.
2. Снова **`ЗАПУСК.bat`** или `npm run go`.

Переустанавливать ничего не нужно. База данных сохраняется в Docker-томе.

## Что уже лежит в архиве (трогать не обязательно)

| Файл / папка | Назначение |
|--------------|------------|
| `config/environment.env` | готовые настройки (копируются в `.env` автоматически) |
| `docker-compose.yml` | PostgreSQL в Docker, порт **5433** |
| `prisma/seed.ts` | демо-пользователь `demo@demo.ru` |
| `ЗАПУСК.bat` | автозапуск для Windows |
| `public/pdfs/` | материалы курсов |

Порт **5433** выбран специально: если на компьютере уже стоит свой PostgreSQL на 5432, конфликта не будет.

## Частые проблемы

### «Docker не найден» / `docker info` с ошибкой

- Установите Docker Desktop.
- Запустите его из меню Пуск и подождите 1–2 минуты.
- Перезагрузите компьютер после первой установки Docker.

### Долго висит на первом запуске

Это нормально: скачиваются образ PostgreSQL (~100 МБ) и пакеты npm. Следующие запуски быстрее.

### Сайт не открылся сам

Откройте вручную: http://localhost:3000

### «Неверный email или пароль»

Используйте демо-аккаунт `demo@demo.ru` / `demo1234` или зарегистрируйтесь заново.

### Порт 3000 занят

Закройте другие окна с `npm run dev` или перезагрузите компьютер.

### Полный сброс базы данных

```powershell
docker compose down -v
npm run go
```

Удалятся все пользователи; демо-аккаунт создастся снова.

### Кнопка внизу слева (логотип Next.js)

Служебная панель режима разработки. На обычном сайте её нет. Можно не нажимать.

## Команды для продвинутых

| Команда | Действие |
|---------|----------|
| `npm run setup` | только настройка (Docker + БД + seed), без сайта |
| `npm run dev` | только сайт (если setup уже делали) |
| `npm run docker:down` | остановить базу |
| `npm run build` | сборка для production |

## Требования

- Windows 10/11, macOS или Linux  
- 8 ГБ RAM и ~2 ГБ свободного места  
- Интернет **только при первом запуске** (скачивание Docker-образа и npm-пакетов)

---

Если что-то не работает — сделайте скриншот окна после `ЗАПУСК.bat` с текстом ошибки.
